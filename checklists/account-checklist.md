# Чек-лист тестирования личного кабинета

## Цель

Проверить авторизацию, регистрацию, восстановление пароля, работу личного кабинета, настройки профиля, заказы и синхронизацию аккаунта на разных устройствах.

## Область проверки

- вход и выход из аккаунта;
- сохранение сессии;
- регистрация;
- восстановление пароля;
- ввод и повторная отправка кода;
- авторизованный личный кабинет;
- разделы профиля;
- заказы;
- параметры профиля;
- синхронизация корзины и избранного;
- работа сессий и токенов на разных устройствах.

---

## 1. Основной функционал

- [ ] Вход с валидным email и паролем проходит успешно.
- [ ] После входа пользователь переходит в личный кабинет.
- [ ] Сессия сохраняется после обновления страницы.
- [ ] Пользователь остается авторизованным после перехода по страницам сайта.
- [ ] Вход работает по нажатию Enter.
- [ ] Пользователь может успешно выйти из аккаунта.
- [ ] После выхода по кнопке “Назад” нельзя увидеть личный кабинет.

## 2. Страница входа и регистрации

- [ ] Страница входа / регистрации открывается.
- [ ] При открытии страницы нет ошибок загрузки.

- [ ] При открытии страницы нет ошибок `500`.
- [ ] При открытии страницы нет ошибок CORS.
- [ ] Кнопка “Войти” отображается и кликабельна.
- [ ] По кнопке “Войти” происходит переход на авторизацию.
- [ ] Кнопка “Зарегистрироваться” отображается и кликабельна.
- [ ] По кнопке “Зарегистрироваться” происходит переход на страницу регистрации.
- [ ] Ссылка “Обратно на сайт” отображается и кликабельна.
- [ ] По ссылке “Обратно на сайт” происходит переход на главную страницу сайта.

## 3. Регистрация

- [ ] При открытии страницы регистрации нет ошибок загрузки.
- [ ] При открытии страницы регистрации нет ошибок `404`.
- [ ] При открытии страницы регистрации нет ошибок `500`.

- [ ] Отображаются все необходимые поля: email, пароль, повтор пароля.
- [ ] Подписи полей понятны.
- [ ] Подсказки полей понятны и корректны.
- [ ] Подсказки исчезают после исправления значения.
- [ ] При пустом email отображается ошибка с подсказкой.
- [ ] При пустом пароле отображается ошибка с подсказкой.
- [ ] При пустых полях отображается ошибка с подсказкой.

- [ ] Ссылка “Назад” отображается и кликабельна.
- [ ] По ссылке “Назад” происходит переход обратно на страницу входа / регистрации.
- [ ] После регистрации вход с введенными данными возможен.
- [ ] Общие подсказки ошибок понятны и соответствуют причине ошибки.

## 4. Поле email при регистрации

- [ ] Поле email является обязательным.
- [ ] Валидный email принимается.
- [ ] Невалидный email не принимается.
- [ ] При пустом или пробельном вводе отображается подсказка.
- [ ] Email без `@` не принимается.
- [ ] Email без домена не принимается.
- [ ] Email с буквами в верхнем регистре принимается.
- [ ] Email с пробелами до или после `@` не принимается.
- [ ] Email с пробелами в начале или конце обрабатывается корректно.
- [ ] Навигационные кнопки “Продолжить” и “Назад” не очищают заполненные поля.

## 5. Поля пароля при регистрации

- [ ] Поле “Пароль” является обязательным.

- [ ] Если один из паролей содержит лишний пробел, отображается ошибка с подсказкой.
- [ ] Пароль из 100 символов обрабатывается корректно.
- [ ] Пароль из 1–5 символов не принимается.
- [ ] При коротком пароле отображается ошибка валидации пароля.
- [ ] Пароль скрыт точками.
- [ ] Кнопка показать / скрыть пароль отображается и кликабельна.
- [ ] Навигационные кнопки “Продолжить” и “Назад” не очищают заполненные поля.

## 15. Синхронизация аккаунта на разных устройствах

### Корзина

- [ ] Без авторизации корзина работает как гостевая в браузере.
- [ ] При авторизации подтягивается корзина пользователя.
- [ ] Пользовательская и гостевая корзины объединяются, если гостевая корзина не пустая.
- [ ] При наличии одинаковых товаров в пользовательской и гостевой корзинах количество суммируется.
- [ ] После logout остается гостевая корзина, если она не пустая.
- [ ] При смене аккаунтов подтягивается корзина соответствующего пользователя.
- [ ] При смене устройств подтягивается корзина соответствующего пользователя.

### Избранное

- [ ] Без авторизации иконки избранного доступны для отметок.
- [ ] При авторизации подтягивается избранное пользователя.
- [ ] Пользовательское и гостевое избранное объединяются без дублей, если товары были отмечены.
- [ ] При logout избранное пользователя становится недоступно.
- [ ] После logout отметки на иконках избранного очищаются.
- [ ] При смене аккаунтов подтягивается избранное соответствующего пользователя.
- [ ] При смене устройств подтягивается избранное соответствующего пользователя.

### Сессии и смена пароля

- [ ] Logout завершает только текущую сессию.
- [ ] После logout другие сессии остаются активными.
- [ ] При смене пароля происходит инвалидация сессий на других устройствах.
- [ ] После смены пароля при оформлении заказа с устаревшей сессией отображается корректная ошибка авторизации.
- [ ] После смены пароля при изменении личных данных с устаревшей сессией отображается корректная ошибка авторизации.
- [ ] При инвалидации сессии подсказка понятна пользователю.
