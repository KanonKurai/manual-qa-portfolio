# Чек-лист регресса поиска, фильтров и сортировки

## Цель

Проверить, что поиск, фильтры, сортировка, пагинация и состояние страницы работают корректно как отдельно, так и в комбинации друг с другом.

## Область проверки

- поиск;
- категории;
- комбинация поиска и фильтров;
- очистка поиска;
- сброс фильтров;
- сортировка;
- пагинация / “Показать ещё”;
- состояние URL;
- возврат из карточки товара;
- пустые состояния;
- консистентность UI и выдачи.

---

## 1. Матрица поведения поиска и фильтров

| Сценарий | Ожидаемое поведение |
|---|---|
| Ввести поисковый запрос | Применяется поиск |
| Выбрать категорию | Применяется фильтр по категории |
| Поиск + категория | Применяются оба условия: поиск и категория |
| Очистить поиск крестиком | Очищается только поисковый запрос, выбранные категории сохраняются |
| Нажать “Сбросить” | Сбрасываются только выбранные фильтры, поисковый запрос сохраняется |
| Пустой поисковый запрос | Отображается каталог товаров, выбранные фильтры сохраняются |
| Пробельный поисковый запрос | Обрабатывается как пустой |
| Обновить страницу | Состояние поиска и фильтров сохраняется |
| Вернуться из карточки товара в каталог | Состояние поиска и фильтров сохраняется |

---

## 2. Базовые сценарии

- [ ] Поиск возвращает корректную выдачу.
- [ ] Очистка поиска крестиком очищает поле поиска.
- [ ] После очистки поиска выдача соответствует текущему состоянию страницы.
- [ ] Нажатие Enter в поиске обновляет выдачу корректно.
- [ ] Выбор одной категории возвращает корректную выдачу.
- [ ] Выбор нескольких категорий возвращает корректную выдачу.
- [ ] При выборе нескольких категорий отображаются товары из выбранных категорий.

---

## 3. Комбинации поиска и категорий

- [ ] Поиск → выбор одной категории: выдача корректна.
- [ ] Выбор одной категории → поиск: выдача корректна.
- [ ] Поиск → выбор нескольких категорий: выдача корректна.
- [ ] Выбор нескольких категорий → поиск: выдача корректна.
- [ ] Поиск + несколько категорий → снять одну категорию: выдача корректна.
- [ ] Поиск + одна категория → сменить категорию на другую: выдача корректна.
- [ ] Поиск + одна категория → снять категорию: остается только поиск, выдача корректна.

---

## 4. Сбросы и очистка

- [ ] Поиск + фильтры → нажать “Сбросить”: сбрасываются фильтры, поиск сохраняется.
- [ ] После сброса фильтров при активном поиске выдача соответствует результатам поиска.
- [ ] Только фильтры → нажать “Сбросить”: фильтры сброшены, отображается полный каталог.
- [ ] Только поиск → нажать “Сбросить”: поиск не сбрасывается, выдача соответствует результатам поиска.
- [ ] Пробельный ввод в поиске обрабатывается как пустой запрос.

---

## 5. Сортировка

- [ ] Сортировка без поиска и фильтров корректно меняет порядок выдачи.
- [ ] Сортировка + поиск: выдача корректна.
- [ ] Сортировка + фильтры: выдача корректна.
- [ ] Сортировка + поиск + фильтры: выдача корректна.

---

## 6. Пагинация / “Показать ещё”

- [ ] Пагинация без поиска и фильтров работает корректно.
- [ ] Пагинация + поиск работает корректно.
- [ ] Пагинация + фильтры работает корректно.
- [ ] Пагинация + поиск + фильтры работает корректно.
- [ ] При смене поиска или фильтра список товаров не смешивается со старой выдачей.

---

## 7. Состояние страницы, URL и история браузера

- [ ] При использовании поиска параметры в URL обновляются корректно.
- [ ] При использовании фильтров параметры в URL обновляются корректно.
- [ ] При очистке поиска параметры в URL обновляются корректно.
- [ ] При сбросе фильтров параметры в URL обновляются корректно.
- [ ] При открытии ссылки с параметрами поиска и категории состояние страницы восстанавливается корректно.
- [ ] При открытии ссылки с параметрами поиска и категории выдача соответствует параметрам.
- [ ] После обновления страницы при активном поиске и фильтрах состояние сохраняется.
- [ ] После перехода в карточку товара и возврата назад состояние поиска и фильтров сохраняется.

---

## 8. Пустые состояния

- [ ] При поиске без результатов отображается сообщение “Ничего не найдено”.
- [ ] При поиске без результатов и выбранной категории отображается сообщение “Ничего не найдено”.
- [ ] При пустой выдаче не подмешиваются товары из старых результатов.

---

## 9. Консистентность UI

- [ ] Поле поиска соответствует текущей выдаче.
- [ ] Выбранные чекбоксы категорий соответствуют текущей выдаче.
- [ ] Выбранная сортировка соответствует текущему порядку товаров.
- [ ] UI не показывает активные фильтры, которые фактически не применены.
- [ ] UI не скрывает активные фильтры, которые фактически применены.
