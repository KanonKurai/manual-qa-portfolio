# Чек-лист API-проверок интернет-магазина

## Цель

Проверить API интернет-магазина: публичные endpoints каталога, товары, рейтинги, авторизацию, профиль пользователя и связь данных с админкой.

## Область проверки

- категории;
- товары;
- фильтрация товаров;
- рейтинги товаров;
- регистрация;
- авторизация;
- logout;
- восстановление пароля;
- профиль пользователя;
- защищенные endpoints;
- обработка невалидных данных;
- проверка токена;
- проверка данных в админке.

## 1. GET /api/categories/

Endpoint не требует авторизации.

- [ ] Запрос возвращает `200 OK`.
- [ ] Без токена запрос возвращает `200 OK`.
- [ ] С токеном запрос возвращает `200 OK`.
- [ ] Ответ приходит в формате JSON.
- [ ] Ответ содержит список категорий.
- [ ] Endpoint работает со слэшем на конце.
- [ ] Endpoint работает без слэша на конце.
- [ ] `POST /api/categories/` возвращает `405 Method Not Allowed`.
- [ ] `GET /api/categories/?page=1` обрабатывается корректно, если пагинация не используется.
- [ ] `GET /api/categories/?search=value` обрабатывается корректно, если фильтрация не используется.

## 3. GET /api/products/

Endpoint не требует авторизации.

### Базовые проверки

- [ ] Получение списка всех товаров работает корректно.
- [ ] Получение новых товаров работает корректно.
- [ ] Получение товаров-бестселлеров работает корректно.
- [ ] Получение товаров по категории работает корректно.
- [ ] Получение новых товаров внутри категории работает корректно.
- [ ] Получение товаров-бестселлеров внутри категории работает корректно.
- [ ] Получение новых товаров-бестселлеров внутри категории работает корректно.
- [ ] Поля рейтинга отображаются корректно.
- [ ] В ответе есть поле среднего рейтинга.
- [ ] В ответе есть поле количества оценок.
- [ ] Без токена endpoint работает корректно.
- [ ] С токеном endpoint работает корректно.
- [ ] При запросе с токеном корректно отображаются данные, зависящие от пользователя.
- [ ] Ответ приходит в формате JSON.
- [ ] Ответ содержит список товаров.
- [ ] Endpoint работает со слэшем на конце.
- [ ] Endpoint работает без слэша на конце.
- [ ] `POST /api/products/` возвращает `405 Method Not Allowed`.

### Фильтрация

- [ ] `GET /api/products/?is_new=true` возвращает новые товары.
- [ ] `GET /api/products/?is_bestseller=true` возвращает товары-бестселлеры.
- [ ] `GET /api/products/?category={category_slug}` возвращает товары выбранной категории.
- [ ] `GET /api/products/?is_new=true&category={category_slug}` возвращает новые товары выбранной категории.
- [ ] `GET /api/products/?is_bestseller=true&category={category_slug}` возвращает бестселлеры выбранной категории.
- [ ] `GET /api/products/?category={category_1}&category={category_2}` обрабатывается корректно.

### Негативные и нестандартные проверки фильтров

- [ ] `GET /api/products/?is_new=false` не применяет фильтр новых товаров и возвращает общий список.
- [ ] `GET /api/products/?is_bestseller=false` не применяет фильтр бестселлеров и возвращает общий список.
- [ ] `GET /api/products/?is_new=abc` не применяет фильтр новых товаров и возвращает общий список.
- [ ] `GET /api/products/?is_bestseller=abc` не применяет фильтр бестселлеров и возвращает общий список.
- [ ] `GET /api/products/?category=abc` возвращает `200 OK` и пустой список.
- [ ] `GET /api/products/?is_new=` возвращает `200 OK` и пустой список.
- [ ] `GET /api/products/?is_bestseller=` возвращает `200 OK` и пустой список.
- [ ] `GET /api/products/?category=` возвращает `200 OK` и пустой список.
- [ ] Неизвестный query-параметр обрабатывается корректно и возвращает `200 OK`.

## 5. POST /api/products/{product_id}/rate/

Endpoint требует авторизации.

- [ ] Запрос на существующий товар с валидным токеном и слэшем на конце возвращает `200 OK`.
- [ ] Запрос на несуществующий товар возвращает `404 Not Found`.
- [ ] Запрос с невалидным `product_id` возвращает `404 Not Found`.
- [ ] Запрос с нечисловым `product_id` возвращает `404 Not Found`.
- [ ] Оценка вне допустимого диапазона возвращает `400 Bad Request`.
- [ ] Запрос без токена возвращает `401 Unauthorized`.
- [ ] Запрос с невалидным токеном возвращает `401 Unauthorized`.
- [ ] Запрос без слэша на конце возвращает `404` или `405`.
- [ ] Запрос без комментария в теле возвращает `200 OK`.
- [ ] Запрос без оценки в теле возвращает `400 Bad Request`.
- [ ] `GET /api/products/{product_id}/rate/` возвращает `405 Method Not Allowed`.

## 6. Успешные сценарии авторизации и профиля

### Регистрация

- [ ] Отправка кода на email для регистрации проходит успешно.
- [ ] Подтверждение email валидным кодом и данными пользователя проходит успешно.
- [ ] После успешной регистрации пользователь получает токен.
- [ ] После успешной регистрации `GET /api/auth/me/` с токеном возвращает `200 OK`.
- [ ] После успешной регистрации аккаунт отображается в админке.

### Авторизация

- [ ] `POST /api/auth/login/` с валидной почтой и паролем возвращает `200 OK`.
- [ ] После успешной авторизации пользователь получает токен.
- [ ] После успешной авторизации `GET /api/auth/me/` с токеном возвращает `200 OK`.

### Logout

- [ ] `POST /api/auth/logout/` с валидным токеном возвращает успешный ответ.
- [ ] После logout `GET /api/auth/me/` с тем же токеном возвращает `401 Unauthorized`.

### Восстановление пароля

- [ ] Отправка кода на email для восстановления пароля проходит успешно.
- [ ] Подтверждение восстановления валидным кодом и новым паролем проходит успешно.
- [ ] После восстановления пароля авторизация с новым паролем возвращает `200 OK`.
- [ ] После восстановления пароля авторизация со старым паролем возвращает ошибку.
- [ ] После восстановления пароля старый токен больше не работает и возвращает `401 Unauthorized`.

### Профиль

- [ ] `GET /api/auth/me/` с валидным токеном возвращает `200 OK`.
- [ ] `PATCH /api/auth/profile/` с валидными данными возвращает `200 OK`.
- [ ] После обновления профиля `GET /api/auth/me/` возвращает актуальные данные.
