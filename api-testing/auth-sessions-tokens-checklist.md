# Чек-лист API-проверок сессий и токенов

## Цель

Проверить, как система обрабатывает несколько активных сессий пользователя, logout и восстановление пароля.

## Область проверки

- авторизация;
- получение токенов;
- проверка активной сессии;
- logout;
- восстановление пароля;
- инвалидация токенов;
- проверка старого и нового пароля.

---

## 1. Logout одной сессии не завершает другие сессии

### Шаги

- [ ] Выполнить `POST /api/auth/login/` и получить токен A.
- [ ] Выполнить `POST /api/auth/login/` повторно и получить токен B.
- [ ] Выполнить `GET /api/auth/me/` с токеном A.
- [ ] Проверить, что ответ `200 OK`.
- [ ] Выполнить `GET /api/auth/me/` с токеном B.
- [ ] Проверить, что ответ `200 OK`.
- [ ] Выполнить `POST /api/auth/logout/` с токеном A.
- [ ] Выполнить `GET /api/auth/me/` с токеном A.
- [ ] Проверить, что ответ `401 Unauthorized`.
- [ ] Выполнить `GET /api/auth/me/` с токеном B.
- [ ] Проверить, что ответ `200 OK`.

### Ожидаемый результат

После logout токена A только эта сессия становится невалидной.  
Токен B остается валидным.

---

## 2. После восстановления пароля другие сессии становятся невалидными

### Шаги

- [ ] Выполнить `POST /api/auth/login/` и получить токен A.
- [ ] Выполнить `POST /api/auth/login/` повторно и получить токен B.
- [ ] Выполнить `GET /api/auth/me/` с токеном A.
- [ ] Проверить, что ответ `200 OK`.
- [ ] Выполнить `GET /api/auth/me/` с токеном B.
- [ ] Проверить, что ответ `200 OK`.
- [ ] Выполнить `POST /api/auth/email/send/` с purpose для восстановления пароля.
- [ ] Выполнить `POST /api/auth/email/verify/` с кодом и новым паролем.
- [ ] Выполнить `GET /api/auth/me/` с токеном A.
- [ ] Проверить, что ответ `401 Unauthorized`.
- [ ] Выполнить `GET /api/auth/me/` с токеном B.
- [ ] Проверить, что ответ `401 Unauthorized`.
- [ ] Выполнить `POST /api/auth/login/` с новым паролем.
- [ ] Проверить, что авторизация проходит успешно.
- [ ] Выполнить `POST /api/auth/login/` со старым паролем.
- [ ] Проверить, что авторизация не проходит.

### Ожидаемый результат

После восстановления пароля все старые токены пользователя становятся невалидными.  
Авторизация доступна только с новым паролем.
