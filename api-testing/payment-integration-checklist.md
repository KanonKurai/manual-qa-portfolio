# Чек-лист API-проверок платежной интеграции

## Цель

Проверить создание платежа, обработку webhook-уведомлений, изменение статусов заказа и платежа, защиту от повторной обработки и некорректных запросов.

## Область проверки

- создание платежа;
- обработка webhook-уведомлений;
- изменение статуса заказа;
- повторная обработка;
- защита от дублей;
- обработка невалидных данных;
- отображение заказа в административной панели.

## 1. Создание платежа

### `POST /api/payments/yookassa/create/`

- [ ] Для заказа с онлайн-оплатой возвращается `200 OK`
- [ ] Для заказа с онлайн-оплатой создаётся платёж
- [ ] Для заказа без онлайн-оплаты возвращается `400 Bad Request`
- [ ] Для заказа без онлайн-оплаты платёж не создаётся
- [ ] При отправке некорректного JSON возвращается `400 Bad Request`
- [ ] При отправке некорректного JSON платёж не создаётся

## 2. Обработка валидных webhook-уведомлений

### `POST /api/payments/yookassa/webhook/`

- [ ] Статус `pending` переводит заказ в состояние «Ожидает оплаты»
- [ ] Статус `succeeded` переводит заказ в состояние «Оплачен»
- [ ] Статус `canceled` переводит заказ в состояние «Отменён»
- [ ] Статус `succeeded` не изменяет уже отменённый заказ
- [ ] Статус `canceled` не откатывает уже оплаченный заказ
- [ ] Повторный `succeeded` не изменяет статус повторно
- [ ] Повторный `canceled` не изменяет статус повторно
- [ ] Повторная отправка webhook не создаёт новый заказ
- [ ] Повторная отправка webhook не создаёт новый платёж

> Часть проверок финальных статусов требует возможности отправки валидного webhook от платёжного провайдера.

## 3. Обработка невалидных webhook-уведомлений

- [ ] Для заказа в статусе `pending` невалидный `succeeded` не изменяет статус
- [ ] Для заказа в статусе `pending` невалидный `canceled` не изменяет статус
- [ ] Для заказа в статусе `succeeded` невалидный `pending` не изменяет статус
- [ ] Для заказа в статусе `succeeded` невалидный `canceled` не изменяет статус
- [ ] Для заказа в статусе `canceled` невалидный `pending` не изменяет статус
- [ ] Для заказа в статусе `canceled` невалидный `succeeded` не изменяет статус
- [ ] Невалидный webhook не создаёт новый заказ
- [ ] Невалидный webhook не создаёт новый платёж

## 4. Негативные проверки webhook

- [ ] При отсутствии `payment_id` возвращается `400 Bad Request`
- [ ] При невалидном `payment_id` возвращается `404 Not Found`
- [ ] При отсутствии `event` возвращается `200 OK`, статус заказа не меняется
- [ ] При противоречии между `event` и `object.status` возвращается `200 OK`, статус заказа не меняется
- [ ] При неизвестном значении `event` возвращается `200 OK`, статус заказа не меняется
- [ ] При отправке некорректного JSON возвращается `400 Bad Request`
- [ ] SQL-инъекция в теле запроса не выполняется
- [ ] Запрос без завершающего слеша обрабатывается корректно
- [ ] Отправка webhook не создаёт новый заказ или платёж

## 5. Административная панель

- [ ] Созданный заказ отображается в разделе «Заказы»
- [ ] Номер заказа отображается корректно
- [ ] Данные пользователя соответствуют данным заказа
- [ ] Состав заказа отображается корректно
- [ ] Стоимость заказа отображается корректно
- [ ] Способ оплаты отображается корректно
- [ ] Статус заказа соответствует результату оплаты
- [ ] Статус платежа соответствует данным платёжного сервиса
