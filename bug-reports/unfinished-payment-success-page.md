# Баг-репорт: после возврата с платежной страницы отображается страница успешного оформления заказа

## Краткое описание

После возврата с платежной страницы отображается страница “Заказ оформлен / Спасибо за заказ”, даже если оплата не была завершена или прошла неуспешно.

При этом статус заказа и платежа остаются корректными. Проблема только в странице результата, которую видит пользователь после возврата.

## Окружение

- Web
- Тестовое окружение
- Desktop / Mobile
- Browser: Chrome / Safari

## Предусловия

- Пользователь оформляет заказ.
- В заказе выбран способ оплаты картой.
- Пользователь перешел на платежную страницу.

## Шаги воспроизведения

1. Оформить заказ.
2. Перейти на платежную страницу.
3. Вернуться на сайт, не завершив оплату.
4. Проверить, какая страница отображается после возврата.

## Фактический результат

После возврата отображается страница “Заказ оформлен / Спасибо за заказ”, хотя оплата не была завершена.

## Ожидаемый результат

При неуспешной или незавершенной оплате страница успешного оформления заказа не отображается.

После возврата пользователь должен видеть корректный результат неуспешной или незавершенной оплаты.

## Влияние

Пользователь видит сообщение об успешном оформлении заказа, хотя оплата не завершена.

Это может ввести пользователя в заблуждение и создать ощущение, что заказ будет обработан, хотя фактически успешной оплаты не было.

## Дополнительно проверено

- Статус заказа после возврата не меняется на “Оплачен”.
- Успешный платеж не создается.
- После перезагрузки страницы статус заказа остается корректным.
- Проблема воспроизводится только на странице результата после возврата с платежной страницы.
