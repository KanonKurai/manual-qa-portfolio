# Баг-репорт: лишний экран перед авторизацией и разное поведение перехода в личный кабинет

## Краткое описание

Неавторизованный пользователь при переходе в личный кабинет попадает на промежуточную страницу с кнопкой “Вход / Регистрация”, вместо прямого перехода к форме авторизации.

На мобильной версии поведение отличается в зависимости от точки входа: через иконку профиля открывается промежуточная страница, а через боковое меню сразу открывается форма входа.

## Окружение

- Web
- Desktop / Mobile
- Browser: Chrome / Safari

## Предусловия

- Пользователь не авторизован.
- На сайте доступны переходы в личный кабинет через иконку профиля и через меню.

## Шаги воспроизведения

### Сценарий 1. Desktop

1. Открыть сайт.
2. Нажать на иконку профиля.
3. Проверить, какая страница открывается.

### Сценарий 2. Mobile, переход через иконку профиля

1. Открыть сайт на мобильном устройстве.
2. Нажать на иконку профиля.
3. Проверить, какая страница открывается.

### Сценарий 3. Mobile, переход через боковое меню

1. Открыть сайт на мобильном устройстве.
2. Открыть боковое меню.
3. Нажать “Личный кабинет”.
4. Проверить, какая страница открывается.

## Фактический результат

На desktop при переходе через иконку профиля открывается промежуточная страница с кнопкой “Вход / Регистрация”.

На mobile поведение отличается:

- через иконку профиля открывается промежуточная страница;
- через боковое меню сразу открывается форма входа.

## Ожидаемый результат

Во всех точках входа неавторизованный пользователь должен попадать сразу на страницу входа или регистрации.

Поведение перехода в личный кабинет должно быть одинаковым на desktop и mobile.

## Влияние

Появляется лишний шаг в ключевом пользовательском сценарии авторизации и регистрации.

Поведение неконсистентно: пользователь получает разный результат в зависимости от точки входа.

Это может ухудшать UX и снижать конверсию во вход или регистрацию.
