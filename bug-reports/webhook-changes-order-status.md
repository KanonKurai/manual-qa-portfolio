# Webhook с противоречивыми данными изменяет статус заказа

## Описание

Webhook-обработчик изменяет статус заказа на основании поля `object.status`, даже если значения `event` и `object.status` противоречат друг другу.

Также вручную сформированный webhook может изменить статус заказа без подтверждения со стороны платёжного сервиса.

## Предусловия

- создан заказ с онлайн-оплатой;
- для заказа создан платёж;
- заказ находится в статусе «Ожидает оплаты»;
- известен `payment_id`.

## Шаги воспроизведения

1. Отправить вручную запрос на webhook-эндпоинт:

```json
{
  "event": "payment.succeeded",
  "object": {
    "id": "<payment_id>",
    "status": "canceled"
  }
}
```

2. Проверить статус заказа.

## Фактический результат

Заказ переходит в статус «Отменён».

## Ожидаемый результат

Статус заказа не изменяется, так как:

- `event` и `object.status` противоречат друг другу;
- webhook не подтверждён платёжным сервисом;
- изменение статуса не должно выполняться только на основании `object.status`.

## Дополнительные проверки

### Вручную отправленный `payment.canceled`

При отправке запроса:

```json
{
  "event": "payment.canceled",
  "object": {
    "id": "<payment_id>",
    "status": "canceled"
  }
}
```

заказ также переходит в статус «Отменён», несмотря на то что webhook отправлен вручную.

### Отмена уже оплаченного заказа

Если отправить webhook со статусом `canceled` для заказа, который уже находится в статусе «Оплачен», заказ переходит в статус «Отменён».

## Риск

Вручную сформированный webhook может:

- изменить статус заказа без подтверждения платёжного сервиса;
- отменить уже оплаченный заказ;
- привести к рассинхронизации статусов между магазином и платёжной системой.
