# Баг-репорт: при серверных ошибках отображаются сообщения о неверных пользовательских данных

## Краткое описание

При авторизации и восстановлении пароля сервер возвращает ошибки `500 Internal Server Error` и `504 Gateway Timeout`.

При этом в интерфейсе отображаются сообщения, которые указывают на ошибку в введенных пользователем данных: email, пароль или код. Это не соответствует фактической причине сбоя.

## Окружение

- Web
- Тестовое окружение
- Desktop / Mobile
- Browser: Chrome / Safari

## Предусловия

- Пользователь находится на странице авторизации или восстановления пароля.
- Сервер возвращает ошибку `500` или `504` при выполнении запроса.

## Шаги воспроизведения

### Сценарий 1. Авторизация

1. Перейти на страницу входа.
2. Ввести email и пароль.
3. Нажать кнопку “Войти”.
4. Проверить ответ сервера в DevTools.
5. Проверить сообщение, которое отображается в интерфейсе.

### Сценарий 2. Восстановление пароля

1. Перейти на страницу восстановления пароля.
2. Ввести email.
3. Нажать кнопку отправки кода.
4. Проверить ответ сервера в DevTools.
5. Проверить сообщение, которое отображается в интерфейсе.

## Фактический результат

Сервер возвращает ошибки:

- `500 Internal Server Error` при восстановлении пароля или регистрации;
- `504 Gateway Timeout` при авторизации.

В интерфейсе отображаются сообщения, которые выглядят как ошибка пользователя:

- “Не удалось отправить код. Проверьте email.”
- “Не удалось выполнить операцию. Проверьте данные.”

## Ожидаемый результат

При серверных ошибках интерфейс должен показывать сообщение, соответствующее фактической причине сбоя.

Например:

- “Ошибка сервера. Попробуйте позже.”
- “Сервис временно недоступен. Повторите попытку позже.”
- “Не удалось выполнить операцию из-за технической ошибки.”

Сообщение не должно указывать на некорректность email, пароля или других пользовательских данных, если проблема произошла на стороне сервера.

## Влияние

Пользователь получает неправильное объяснение ошибки и может думать, что проблема в его email, пароле или введенных данных.

Это может привести к повторным безуспешным попыткам, потере доверия к сервису и обращениям в поддержку.

## Дополнительно проверено

- В DevTools в Network видны ответы сервера с кодами `500` и `504`.
- Сообщение в интерфейсе не соответствует HTTP-статусу ответа.
- Ошибка воспроизводится в сценариях авторизации и восстановления пароля.
