# Баг-репорт: 502 Bad Gateway при создании платежа после включения проверки чеков

## Краткое описание

После включения режима проверки чеков в тестовом магазине платежного сервиса запрос на создание платежа возвращает ошибку `502 Bad Gateway`.

До включения проверки чеков основные сценарии оплаты работали корректно. После включения режима проверки чеков пользователь не может перейти к оплате, поэтому проверить формирование чеков невозможно.

## Окружение

- Web
- Тестовое окружение
- Тестовый магазин платежного сервиса
- Оплата банковской картой

## Предусловия

- В тестовом магазине платежного сервиса включен режим проверки чеков.
- На сайте доступно оформление заказа.
- В корзине есть товар.

## Шаги воспроизведения

1. Открыть сайт.
2. Добавить товар в корзину.
3. Перейти к оформлению заказа.
4. Заполнить обязательные данные.
5. Нажать кнопку перехода к оплате.

## Фактический результат

Запрос на создание платежа возвращает `502 Bad Gateway`.

Платеж не создается.  
Переход на страницу оплаты не выполняется.  
Проверить формирование чеков невозможно.

## Ожидаемый результат

После включения режима проверки чеков создание платежа должно работать корректно.

Пользователь должен перейти на страницу оплаты, а после успешной оплаты должен сформироваться чек.

## Влияние

Пользователь не может перейти к оплате.  
Платежный сценарий блокируется на этапе создания платежа.  
Проверка чеков невозможна, потому что платеж не создается.

## Дополнительно для анализа

Нужно проверить:

- реализована ли на backend передача данных для чека;
- какие данные для чека отправляются в платежный сервис;
- все ли обязательные поля для формирования чека передаются корректно;
- почему запрос на создание платежа возвращает `502 Bad Gateway`;
- появляется ли ошибка только после включения проверки чеков.

## Приоритет

High

## Severity

Major
