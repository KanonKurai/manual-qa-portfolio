# Сравнение вариантов регистрации для интернет-магазина

## Цель

Сравнить несколько вариантов регистрации с точки зрения пользователя, бизнеса и сложности реализации.  
Документ подготовлен как пример продуктового анализа со стороны QA.

---

## 1. Email + пароль + подтверждение почты

### Плюсы для пользователя

- Привычный и понятный сценарий регистрации.
- Есть контроль доступа через почту.
- Можно восстановить пароль.

### Минусы для пользователя

- Нужно создавать и помнить пароль.
- Нужно подтверждать почту.

### Плюсы для бизнеса

- Относительно простая и недорогая реализация.
- Ниже сложность поддержки на старте.
- Хорошо подходит для MVP.

### Минусы для бизнеса

- Возможны регистрации на фейковые почты.
- Нужна базовая защита от ботов и злоупотреблений.

---

## 2. Google / OAuth login

### Плюсы для пользователя

- Быстрый вход.
- Не нужно придумывать пароль.
- Меньше шагов при регистрации.

### Минусы для пользователя

- Аккаунт привязывается к стороннему сервису.

### Плюсы для бизнеса

- Может повысить конверсию регистрации.
- Меньше проблем с забытыми паролями.
- Меньше нагрузки на сценарии восстановления доступа.

### Минусы для бизнеса

- Зависимость от стороннего провайдера.
- Дополнительная интеграция и поддержка.

---

## 3. SMS-авторизация

### Плюсы для пользователя

- Не нужно помнить пароль.
- Вход через телефон часто воспринимается как более защищенный.

### Минусы для пользователя

- Нужно указывать номер телефона.
- Возможны задержки или проблемы с доставкой кода.

### Плюсы для бизнеса

- Более строгая верификация пользователя.
- Меньше фейковых аккаунтов.

### Минусы для бизнеса

- Дополнительные расходы на SMS.
- Интеграция стороннего сервиса.
- Выше сложность поддержки.

---

## Вывод

Для MVP оптимальным вариантом выглядит регистрация через email, пароль и подтверждение почты.

Такой сценарий проще и быстрее реализовать, он дешевле в поддержке и привычен пользователям.

Телефон или OAuth можно добавить позже как дополнительные способы входа и регистрации, а не как замену email. Так пользователь сможет выбрать удобный вариант, а риск потери части аудитории будет ниже.
